# 第 12 章：在心理学中使用模型 (Using Models in Psychology)

在前几章中，我们学习了如何构建模型、估计参数以及比较模型。本章将退后一步，探讨一个更根本的问题：**我们如何利用模型来理解人类的心理和行为？**

建模不仅仅是拟合数据，它是一种思维工具，帮助我们探索理论的边界，验证假设的必要性，并促进科学交流。

## 1. 理论背景 (Theory)

### 1.1 建模的流程
心理学研究通常从理论到实验再到数据。建模在其中扮演了桥梁的角色：
1.  **理论 (Theory)**：通常是语言描述的，可能比较模糊。
2.  **模型 (Model)**：将理论转化为具体的计算实现。这包括**核心假设**（理论的关键部分）和**辅助假设**（为了让模型跑起来而做的假设，如特定的数学形式）。
3.  **预测 (Predictions)**：模型在特定参数下产生的行为。
4.  **数据 (Data)**：人类在实验中的行为。

我们的推断建立在模型预测与数据的匹配程度上。

### 1.2 从模型中得出结论
我们可以通过多种方式从模型中学习：

*   **模型探索 (Model Exploration)**：即使没有数据，仅仅运行模型也能帮助我们检查推理是否正确。有时模型会表现出反直觉的行为（例如，分散注意力反而降低了焦点的概率估计），这能纠正我们的直觉。
*   **分析模型 (Analyzing the Model)**：通过操纵参数来隔离特定过程的贡献。例如，Lewandowsky (1999) 通过调节“反应抑制”参数，证实了它与序列回忆中的近因效应（Recency Effect）有关。
*   **参数恢复 (Parameter Recovery)**：在收集数据前，先用已知参数生成模拟数据，再尝试把参数估计回来。如果做不到，说明模型或实验设计有问题。

## 2. 核心概念：充分性与必要性

### 2.1 充分性 (Sufficiency)
**定义**：展示一个模型**能够**解释某些现象。
**意义**：这是一个较弱的主张，因为可能有其他模型也能解释。但在某些情况下，如果一个模型起初看起来不太可能成功，但最终成功了，这就很有意义。

*   **案例 1：PDP 阅读模型** (Seidenberg & McClelland, 1989)。一个没有显式规则的连接主义网络，竟然能模拟出单词阅读中的规则效应（如规则词读得快）。这表明“规则行为”不一定需要“规则机制”。
*   **案例 2：Remember-Know 区分**。通常认为“记得”(Remember) 和“知道”(Know) 对应两个不同的记忆系统。但 Dunn (2004) 展示了一个简单的单维度信号检测论模型也能解释这些数据。这挑战了双加工理论的必要性。

### 2.2 必要性 (Necessity)
**定义**：展示某个假设是解释数据所**必须**的。
**方法**：
1.  **模型比较**：通过贝叶斯因子等方法，证明模型 A 比模型 B 好得多。
2.  **局部必要性**：在模型内部“关闭”某个机制（如将参数设为 0），看模型拟合是否显著变差。例如，Lewandowsky et al. (2004) 发现关闭“输出干扰”会显著降低模型拟合，从而证明了输出干扰的必要性。

### 2.3 逼真度 (Verisimilitude) vs. 真理
所有模型都是错的。但这不代表它们没用。
*   **逼真度**：指一个理论接近真理的程度。即使模型在细节上是错的（如撞车报道中弄错了司机的中间名），它仍然比一个完全胡编乱造的报道（完全错误的模型）更有“逼真度”。
*   **累积信用**：如果一个模型能做出反直觉且被证实的预测，它就积累了信用（Money in the bank），值得我们继续使用和修正。

## 3. 实践指南：可重复性与代码共享

本章没有具体的数学模型代码，而是强调了**科学工作流 (Scientific Workflow)** 的重要性。为了让模型真正成为科学交流的工具，我们需要遵循良好的实践。

### 3.1 增强可重复性的建议

1.  **使用纯文本 (Use Plain Text)**：
    *   代码和数据尽量保存为纯文本格式（如 `.R`, `.csv`, `.txt`），而不是二进制格式（如 `.docx`, `.mat`）。纯文本跨平台、跨时代，易于版本控制。

2.  **合理的命名 (Sensible Naming)**：
    *   变量名应具有描述性（如 `learning_rate` 而非 `x`）。
    *   避免使用系统保留字（如在 MATLAB 中不要用 `i` 做变量名，因为它代表虚数单位）。

3.  **使用调试器 (Use the Debugger)**：
    *   不要只靠 `print` 语句。学会使用 RStudio 或 MATLAB 的断点 (Breakpoint) 和单步执行功能。
    *   **无报错 $\neq$ 正确**。必须用已知结果的简单案例来测试代码逻辑。

4.  **版本控制 (Version Control)**：
    *   强烈推荐使用 **Git**。它能记录代码的每一次修改，让你能随时回退到之前的版本。
    *   使用分支 (Branch) 来尝试新想法，而不破坏主代码。

5.  **代码共享 (Sharing Code)**：
    *   将代码上传到 GitHub 或 Bitbucket。
    *   这不仅是为了别人，也是为了未来的自己。
    *   包含 `README.txt`，说明如何运行代码。

### 3.2 为什么这很重要？
Amy Perfors 在 "In Vivo" 部分指出，很多建模论文让人看不懂，是因为作者没有解释清楚**为什么**模型能工作。
*   清晰的解释不仅仅是列出方程。
*   需要解释哪些假设是核心的，哪些是辅助的。
*   需要展示模型在不同参数下的行为范围。

## 4. 总结

建模是心理学研究中强大的工具。它迫使我们明确理论假设，并提供了检验这些假设的定量方法。虽然所有模型都是对现实的简化（因此是“错”的），但只要它们能提供洞见、做出预测并促进理解，它们就是有价值的。

通过遵循良好的编程和共享规范，我们可以让模型成为科学共同体的共享知识，而不仅仅是个人电脑里的黑箱。
